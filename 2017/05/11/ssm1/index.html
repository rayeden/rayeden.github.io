2、用户登录示例

时序图

业务逻辑很简单，用户发起一个登录请求，传递用户名和密码到服务端后台，后台由Service层接受请求后进行业务处理，发送给dao层进行数据交互处理，dao层由mybatis的配置文件直接和数据库进行交互，获取数据库数据，把结果返回给Service。

3、数据库建表

CREATE DATABASE xssm;
DROP TABLE IF EXISTS user;
CREATE TABLE user(
     id INT NOT NULL AUTO_INCREMENT,
     userName VARCHAR(255) NOT NULL,
    password VARCHAR(255) NOT NULL,
     createTime DATE NOT NULL,
      PRIMARY KEY(id)
)ENGINE=InnoDB DEFAULT CHARSET=utf8;
